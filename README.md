# vaibe-os — шаблон

Операционная система «второго мозга» на движке LLM: управление знаниями, проектами и развитием через AI-ассистентов в Cursor.

## Как использовать этот шаблон

1. **Клонируйте репозиторий** (или используйте его как основу для своего инстанса):
   ```bash
   git clone https://github.com/vAIbe-studio/vaibe-os.git my-vaibe-os
   cd my-vaibe-os
   ```

2. **Добавьте свой контент:**
   - Создайте проекты в `Проекты/{Название проекта}/` (см. структуру ниже).
   - Добавьте роли в `Роли/{Название роли}/` с `README.md` и при необходимости `База знаний/`.
   - При необходимости заполняйте корневую `База знаний/` и подключайте субмодули в `repositories/`.

3. **Работайте в Cursor:** откройте папку как workspace; команды (`.cursor/commands/`) и правила (`.cursor/rules/`) подхватываются автоматически.

## Что входит в шаблон

- **Структура папок** (пустые заглушки): `Проекты/`, `Роли/`, `База знаний/`, `Инбокс/`, `repositories/`, `tools/`.
- **Правила системы:** `.cursor/rules/` — структура файлов, Git (кросс-платформенность), роли, контент и др.
- **Команды:** `.cursor/commands/` — `/inbox-check`, `/task-create`, `/task-execute`, `/plan-update`, `/evolve` и др.
- **Утилиты:** `tools/` — пустая папка с индексом `TOOLS_INDEX.md`; добавляйте скрипты по мере необходимости.
- **Конфигурация:** `.env.example` — шаблон переменных окружения (YouTrack, Cursor и др.); скопируйте в `.env` и заполните.

Подробные правила структуры и именования — в `.cursor/rules/structure.mdc`.

## Основные команды

| Команда | Назначение |
|--------|------------|
| `/inbox-check` | Импорт файлов из `Инбокс/` в структуру |
| `/task-create` | Создание задачи в проекте |
| `/task-execute` | Выполнение задачи по плану с подтверждениями |
| `/plan-update` | Обновление плана проекта |
| `/evolve` | Фиксация инсайтов и улучшение системы |

## Обновление инстанса из шаблона

Если вы создали инстанс на основе этого репозитория и хотите получать обновления шаблона:

```bash
git remote add upstream https://github.com/vAIbe-studio/vaibe-os.git
git fetch upstream
git merge upstream/main
```

При конфликтах приоритет отдавайте версиям из шаблона для файлов в `.cursor/` и корневой структуре.

## Папка `tools/` и `.env`

- **tools/:** в шаблоне есть пустая папка с `TOOLS_INDEX.md`. Добавляйте сюда утилиты (конвертация, отчёты, интеграции) и обновляйте индекс.
- **.env:** скопируйте `.env.example` в `.env`, подставьте свои значения. Файл `.env` в `.gitignore` — не коммитить.
