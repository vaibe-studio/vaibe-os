# Purpose

Quickly qualify leads into ABCDX segments to focus sales effort on highest-value customers and avoid spending resources on unprofitable ones.

# When to use

- Evaluating new lead or customer segment
- Building ICP (Ideal Customer Profile)
- Prioritizing sales pipeline
- Preparing segmentation table for a product/market

# Procedure

## Step 1 — Classify into segments

| Segment | Profile | Action |
|---|---|---|
| **A** — Ideal | High value, fast decisions, no alternatives, critical need | Scale rapidly, priority support |
| **B** — Promising | Real need but objections, medium cycle, alternatives exist | Build cases, address objections, demonstrate advantages |
| **C** — Low-value | Low value, many objections, high resource cost | Minimize effort, standardize, consider dropping |
| **D** — Non-paying | Many questions, never buy (no real problem or no budget) | Qualify fast, filter early, standard responses |
| **X** — Potential | Attractive segment but no solution/competence yet | Plan product development, assess investment |

## Step 2 — Score using 7 criteria

1. **Product value** — how critical is the problem solved?
2. **Decision speed** — typical deal cycle length
3. **Payment size** — ACV/ARR/margin
4. **Alternatives** — competitors/in-house/substitutes
5. **Customization need** — development effort required
6. **Budget availability** — confirmed budget and decision-maker
7. **Growth potential** — expansion within company/geography

## Step 3 — Build value proposition per segment

For each segment, document:
- Problem criticality and scale
- Alternatives and their limitations
- ROI (quantified if possible)
- Resource savings (time/money/FTE)
- Risk of inaction

## Step 4 — Prepare case studies (for A and B)

A case study is ready when:
- Real customer from the segment exists
- Product is deployed and used
- Measurable results available
- Concrete benefit demonstrated (in money or metrics)

Use cases to: reduce perceived risk, argue with numbers, show social proof.

# Output format

Segmentation table with columns:

| Column | Description |
|---|---|
| Customer segment | Description relevant to sales |
| ABCDX category | A / B / C / D / X |
| Pain/problem | What hurts |
| Solution | What exactly the product closes |
| Value/benefit | Measurable |
| Case study | Yes/No + link |

# Quality bar

- [ ] All segments classified with clear criteria
- [ ] Focus on A and B segments justified
- [ ] Quantitative value where possible
- [ ] Alternatives known and compared
- [ ] Arguments prepared for B segment objections

# Anti-patterns

- Treating all segments equally without prioritization
- Missing budget/decision-maker qualification for D leads
- Qualitative-only value propositions without metrics
- Mixing segments without explicit scoring criteria
