# Структура vAIbe-OS (краткая версия)

> Полная версия: `.ai/rules/structure.md`

## Ключевые пути

| Путь | Назначение |
|---|---|
| `Проекты/{NAME}/` | Проект с `README.md` (обяз. поле `Видимость: командный\|личный`) |
| `Проекты/{NAME}/Задачи/{NUM}-{TITLE}/` | Задача; `task.md` с `## Статус` в конце |
| `Проекты/{NAME}/Задачи/.../results/v{N}/` | Версионированные результаты |
| `Проекты/{NAME}/Встречи/Встреча {N}. {DD.MM.YYYY} - {TITLE}/` | Встреча: `transcript.md`, `summary.md`, `tasks.md` |
| `Проекты/{NAME}/Исходные материалы/` | Документы, спецификации, макеты |
| `Проекты/{NAME}/База знаний/` | Накапливаемые знания проекта |
| `Проекты/{NAME}/Планы/` | (опц.) Snapshot-планы: `План - {N} - {DD.MM.YYYY} - {TITLE}.md` |
| `База знаний/` | Персональные знания пользователя |
| `Контакты/` | Справочник персоналий (один человек = один .md) |
| `Инбокс/` | Входящие файлы для импорта (`/inbox-check`) |
| `tools/` | Python-утилиты (`python -m tools.{name}`) |
| `repositories/` | Git submodules с кодом |

## Обязательные подпапки проекта

`README.md`, `Задачи/`, `Встречи/`, `Исходные материалы/`, `База знаний/`

## Формат task.md — секция `## Статус`

Единая точка правды, **всегда в конце** файла:

```markdown
## Статус
- **Статус**: открыта | в процессе | выполнена | на холде
- **Завершена**: YYYY-MM-DD
- **Версия результата**: v{N} | inline | inline (описание)
```

## Версионирование результатов

- `results/v1/`, `results/v2/`, ... — каждое выполнение в отдельной подпапке
- Предыдущие версии не затираются
- Legacy-файлы напрямую в `results/` — допустимы, при новом выполнении создать `v1/` + `v2/`

## .ai/ — IDE-independent ядро

```
.ai/
├── ONTOLOGY.md          — онтологический фундамент (5 законов)
├── MANIFESTO.md         — поведенческие принципы
├── MANIFESTO-brief.md   — краткая версия для auto-loading
├── GUARDS.md            — формализованные инварианты
├── router.md            — маршрутизация задач → skills
├── rules/               — канонические правила (source of truth)
├── skills/              — actionable playbooks
└── knowledge/           — справочные материалы
```

## Правила именования

1. Папки и файлы — **на русском языке** (кроме `tools/`, `.ai/`, `repositories/`, системные `.md`)
2. PascalCase для русских названий
3. Номера задач: ведущие нули — `001`, `002`, `010`
4. **ЗАПРЕЩЕНО** менять только регистр в имени (конфликт Linux/Windows). См. `.ai/rules/git-cross-platform.md`

## Видимость проектов

- `командный` (по умолчанию) — трекается в Git
- `личный` — добавить `/Проекты/{NAME}/` в `.gitignore`
- Фиксируется в `README.md` проекта
