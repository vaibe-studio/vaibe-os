# Работа с творческим контентом: защита от фильтрации

## Проблема

Серверные фильтры контента (Azure OpenAI, Anthropic и др.) могут блокировать ответ модели, если в контексте запроса накапливается «чувствительный» контент: тексты песен с темами насилия, смерти, саморазрушения, мата и т.д.

Cursor автоматически прикрепляет к запросу:
- Открытые файлы
- Недавно просмотренные файлы
- Файлы, указанные через `@`

Когда несколько файлов переводов с «тяжёлыми» темами попадают в контекст одновременно — фильтр срабатывает.

Ошибка выглядит так:
```
Output blocked by content filtering policy
```

## Правила для агента

### 1. Минимизируй контекст

При работе с переводами песен и другим творческим контентом:
- **НЕ читай** файлы других переводов, если они не нужны для текущей задачи
- **НЕ прикрепляй** несколько файлов переводов к одному запросу
- Если нужен шаблон/формат — извлеки только структуру (заголовки, формат таблиц), не содержание

### 2. Работай по одному

- Один перевод за один чат
- Если пользователь просит перевести новую песню — работай только с ней, не загружая предыдущие переводы для «справки»

### 3. Разбивай на части

- Если перевод объёмный — работай по блокам (куплет, припев, бридж), а не генерируй весь файл целиком
- Это снижает объём потенциально «чувствительного» контента в одном ответе

### 4. Формулируй нейтрально

- Не дублируй «чувствительные» строки оригинала в своих комментариях/объяснениях
- В смысловой карте используй аналитический язык, а не цитирование «тяжёлых» строк

### 5. При срабатывании фильтра

Если фильтр всё же сработал:
1. Предложи пользователю **закрыть другие файлы переводов** в IDE (они попадают в контекст как «recently viewed»)
2. Предложи **начать новый чат**
3. Предложи **сменить модель** (разные модели имеют разные фильтры)
4. Попробуй **переформулировать** запрос или разбить на части

## Применение

Это правило применяется при работе с файлами в:
- `База знаний/Творчество/`
- Любыми файлами, содержащими тексты песен, стихов и другого художественного контента
