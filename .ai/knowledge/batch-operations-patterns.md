# Паттерны массовых операций

Справочник по выполнению batch-операций (массовое обновление файлов, задач, проектов) в vAIbe-OS. Основан на практическом опыте массовой стандартизации задач по множеству проектов.

## Главное правило

> После любой массовой операции — обязательный **полный enumerate + verify** каждого элемента.

Не доверять ни своей памяти, ни субагентам, ни скриптам — всегда проверять финальный результат программно.

## Паттерн: полная проверка

```
1. Получить ПОЛНЫЙ список элементов (Get-ChildItem / Glob)
2. Для КАЖДОГО проверить целевое условие (секция есть? файл на месте?)
3. Сравнить: total == ok? Если нет — отчёт issues
```

Пример (PowerShell, проверка `## Статус` во всех task.md):
```powershell
[Console]::OutputEncoding = [System.Text.Encoding]::UTF8
Get-ChildItem -Path ".\Проекты" -Recurse -Filter "task.md" | ForEach-Object {
    $content = Get-Content $_.FullName -Raw -Encoding UTF8
    if ($content -notmatch "## Статус") { Write-Output "MISSING: $($_.FullName)" }
}
```

## Типичные ошибки

### 1. Пропуск проектов при параллельных субагентах

**Проблема**: при распределении работы между 4 субагентами один пропустил проект из-за encoding/path проблем и не сообщил об ошибке.

**Решение**: после завершения всех субагентов — запускать единый скрипт верификации, который перечисляет ВСЕ элементы и проверяет каждый.

### 2. Забыть проект при ручном перечислении

**Проблема**: при ручном распределении задач по субагентам упустили vAIbe-OS (17-й проект из 17).

**Решение**: всегда начинать с `Get-ChildItem -Path ".\Проекты" -Directory`, записать полный список, потом работать по нему.

### 3. Не проверить edge cases

**Проблема**: задачи без `results/`, задачи с inline-результатами, задачи в архиве — все требуют разных шаблонов `## Статус`.

**Решение**: перед массовой операцией составить таксономию вариантов и обработать каждый.

## Рекомендации по параллелизации

- Максимум 4 субагента одновременно (ограничение системы)
- Каждому субагенту — явный список проектов/задач (не «остальные»)
- После завершения — единая верификация вне субагентов
- Субагенты с моделью `fast` — для массовых однотипных правок (добавить секцию, скопировать файл)
