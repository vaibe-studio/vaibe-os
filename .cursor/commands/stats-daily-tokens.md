# Статистика токенов Cursor за день

## Обзор

Команда `/stats-daily-tokens` получает и отображает статистику расхода токенов Cursor за текущий день (или указанную дату).

---

## Формат вызова
- `/stats-daily-tokens` (запускает `python -m tools.cursor_stats`)
- `/stats-daily-tokens --date YYYY-MM-DD`
- `/stats-daily-tokens --raw`
- `/stats-daily-tokens -v`

## Входы (источники данных)
- `.env` (локально): `CURSOR_AUTH_COOKIE`
- `tools/cursor_stats/` — скрипт и зависимости

## Выходы (артефакты)
- Markdown-отчёт в выводе команды (в чат)

---

## Критические правила
> **ЗАПРЕЩЕНО**: публиковать/коммитить значение `CURSOR_AUTH_COOKIE` (см. “Безопасность” ниже).
>
> **ОБЯЗАТЕЛЬНО**: если cookie не настроен или истёк — остановиться и выдать инструкцию, не пытаясь “угадывать”.

---

## Требования

Для работы команды необходимо:

1. **Установленные зависимости**: `pip install -r tools/cursor_stats/requirements.txt`
2. **Настроенный CURSOR_AUTH_COOKIE** в файле `.env`

### Получение CURSOR_AUTH_COOKIE

1. Откройте https://cursor.com в браузере
2. Войдите в свой аккаунт Cursor
3. Откройте DevTools (F12) → Application → Cookies → cursor.com
4. Найдите cookie `WorkosCursorSessionToken`
5. Скопируйте его значение
6. Добавьте в `.env` файл: `CURSOR_AUTH_COOKIE=скопированное_значение`

---

## Алгоритм

### 1. Проверка конфигурации

Проверить наличие `CURSOR_AUTH_COOKIE` в `.env` файле.

**PowerShell (Windows):**

```powershell
Select-String -Path .env -Pattern '^CURSOR_AUTH_COOKIE='
```

**Bash (Linux/macOS, Git Bash):**

```bash
rg '^CURSOR_AUTH_COOKIE=' .env
```

Если переменная не настроена, вывести инструкцию по настройке из раздела "Получение CURSOR_AUTH_COOKIE" выше.

### 2. Запуск скрипта

Выполнить Python-скрипт для получения статистики:

```bash
python -m tools.cursor_stats
```

### 3. Вывод результатов

Скрипт выводит отформатированный Markdown-отчёт с:

- **Резюме**: общее количество токенов, стоимость, число запросов
- **По моделям**: разбивка использования по каждой модели
- **Детальная таблица**: все запросы за день с временем, моделью, токенами и стоимостью

Вывести результат пользователю как есть (это уже Markdown).

## Параметры команды

| Параметр | Описание | Пример |
|----------|----------|--------|
| `--date` | Дата для получения статистики (YYYY-MM-DD) | `--date 2026-01-27` |
| `--raw` | Вывести сырые CSV данные | `--raw` |
| `-v` | Подробный вывод для отладки | `-v` |

## Примеры

```bash
# Статистика за сегодня
python -m tools.cursor_stats

# Статистика за конкретную дату
python -m tools.cursor_stats --date 2026-01-25

# Сырые данные для анализа
python -m tools.cursor_stats --raw
```

## Обработка ошибок

### Если CURSOR_AUTH_COOKIE не настроен

Вывести пользователю инструкцию:

```
❌ CURSOR_AUTH_COOKIE не настроен.

Для настройки:
1. Откройте cursor.com в браузере и войдите в аккаунт
2. DevTools (F12) → Application → Cookies → cursor.com
3. Скопируйте значение WorkosCursorSessionToken
4. Добавьте в .env: CURSOR_AUTH_COOKIE=ваше_значение
```

### Если авторизация не прошла (401)

Cookie мог истечь. Предложить пользователю обновить CURSOR_AUTH_COOKIE по инструкции выше.

### Если нет данных за день

Сообщить, что за выбранную дату нет записей об использовании.
