# Получение обратной связи из YouTrack

## Обзор
Команда `/yt-project-tasks-pull` получает обратную связь из YouTrack в vAIbe-os. Это **вторичный поток синхронизации**: YouTrack → vAIbe-os. Сравнивает статусы задач в YouTrack с последним планом проекта и создаёт новую версию плана с отражением изменений.

**ВАЖНО**: Проект должен быть предварительно связан через `/yt-project-link`.

---

## Формат вызова
- `/yt-project-tasks-pull`
- `/yt-project-tasks-pull {PROJECT_NAME}`

## Входы (источники данных)
- `.env` (локально): `YOUTRACK_URL`, `YOUTRACK_TOKEN`
- `Проекты/{PROJECT}/README.md` — связь с YouTrack (секция YouTrack Integration)
- `Проекты/{PROJECT}/Планы/План - N - ...md` — последний snapshot-план (с чем сравниваем)

## Выходы (артефакты)
- Новый файл плана в `Проекты/{PROJECT}/Планы/` (версия N+1) с:
  - секцией “Изменения с версии N”
  - обновлёнными статусами/комментариями

---

## Канон данных (важно)
- `Проекты/{PROJECT}/Задачи/*/task.md` — канон детальной карточки задачи.
- `Проекты/{PROJECT}/Планы/*` — snapshot управления и интеграций.

`/yt-project-tasks-pull` по умолчанию **не редактирует** карточки задач, а отражает изменения в новой версии плана (чтобы избежать неожиданных правок). При необходимости агент может предложить отдельным шагом синхронизировать статусы в `task.md` (только после явного подтверждения).

---

## КРИТИЧЕСКИЕ ПРАВИЛА

> **ОБЯЗАТЕЛЬНО**: Работать с папкой `Планы/`, а не `Задачи/`.
> **ОБЯЗАТЕЛЬНО**: Сравнивать статусы YouTrack с последним планом.
> **ОБЯЗАТЕЛЬНО**: Создавать новый план с указанием изменений.
> **ОБЯЗАТЕЛЬНО**: Показать сводку изменений перед созданием плана.
> **ЗАПРЕЩЕНО**: Изменять существующие планы — только создавать новые версии.

---

## Концепция

Pull получает **input от людей**, работающих через YouTrack:
- Менеджеры меняют статусы задач
- Тестировщики закрывают баги
- Коллеги отмечают выполнение своих задач
- Команда оставляет комментарии

**Результат**: Новая версия плана в папке `Планы/` с:
- Обновлёнными статусами задач
- Секцией "Изменения с версии N"
- Историей версий плана

## Алгоритм

### 1. **Инициализация**
   - Пользователь вызывает `/yt-project-tasks-pull`
   - Опционально указывает проект: `/yt-project-tasks-pull {PROJECT_NAME}`

### 2. **Проверка конфигурации**
   
   ```
   🔍 Проверка конфигурации...
   
   ✅ YOUTRACK_URL: https://yt.example.com
   ✅ YOUTRACK_TOKEN: настроен
   ✅ Проект связан: ENSO
   ✅ Последний план: План - 1 - 27.01.2026 - MVP готов.md
   ```

### 3. **Поиск последнего плана**

   Агент находит последний план в папке `Проекты/{PROJECT}/Планы/`:
   - Сортировка по номеру версии (План - **N** - ...)
   - Чтение метаданных синхронизации из плана
   - Извлечение статусов задач из таблиц плана

### 4. **Запрос данных из YouTrack**
   
   ```
   🔄 Запрос данных из YouTrack...
   
   Проект: ENSO
   Сравнение с: План - 1 - 27.01.2026 - MVP готов.md
   ```
   
   API запрос:
   ```
   GET /api/issues?query=project:{PROJECT_ID}&fields=id,idReadable,summary,customFields(name,value(name))
   ```

### 5. **Сравнение статусов**
   
   Сравнение статусов YouTrack с последним планом:
   
   | Статус YouTrack | Статус в плане | Результат |
   |-----------------|----------------|-----------|
   | Open | ⬜ | Без изменений |
   | In Progress | ⬜ | Изменение: ⬜ → 🔄 |
   | Fixed | ⬜ | Изменение: ⬜ → ✅ |
   | Verified | ⬜ | Изменение: ⬜ → ✅ |
   | Fixed | ✅ | Без изменений |

### 6. **Получение комментариев**

   Для каждой задачи получить комментарии:
   ```
   GET /api/issues/{issueId}/comments?fields=id,text,created,author(login,fullName)
   ```

### 7. **Показ сводки изменений**

> ⚠️ **ТОЧКА ОСТАНОВА**: Показать изменения и запросить подтверждение.

   ```
   📬 Обновления из YouTrack
   
   📁 Проект: ЭНСО (ENSO)
   📄 Сравнение с: План - 1 - 27.01.2026 - MVP готов.md
   
   ────────────────────────────────────────────────────────
   🔄 Изменения статусов (2)
   ────────────────────────────────────────────────────────
   
   1. ENSO-11 — Ревью опросника для клиента
      📊 Open → Verified (⬜ → ✅)
   
   2. ENSO-12 — Отправка опросника Ксении
      📊 Open → In Progress (⬜ → 🔄)
   
   ────────────────────────────────────────────────────────
   💬 Новые комментарии (1)
   ────────────────────────────────────────────────────────
   
   1. ENSO-13 — Формирование КП
      👤 @manager (14:30):
      "Добавить в КП блок про анализ звонков"
   
   ────────────────────────────────────────────────────────
   
   Создать новый план с этими изменениями?
   1. Да, создать План - 2
   2. Только просмотреть (без изменений)
   3. Отменить
   ```
   
   **→ СТОП. Ожидание выбора.**

### 8. **Создание нового плана**

   При подтверждении создаётся новый план:
   
   **Имя файла**: `План - {N+1} - {ДАТА} - {КРАТКОЕ_ОПИСАНИЕ}.md`
   
   Где `{КРАТКОЕ_ОПИСАНИЕ}` — автоматически генерируется на основе изменений:
   - "Обновление статусов из YouTrack"
   - "ENSO-11 выполнена"
   - и т.д.

### 9. **Структура нового плана**

   Новый план содержит:
   
   1. **Секция "Изменения с версии N"** — в начале документа
   2. **Обновлённые статусы** — в таблицах задач
   3. **История версий** — в конце документа
   4. **Обновлённые метрики** — пересчитанные проценты выполнения

### 10. **Итоговый отчёт**
   
   ```
   ✅ Pull завершён
   
   📊 Результаты:
   - Изменений статусов: 2
   - Новых комментариев: 1
   
   📄 Создан план:
   Проекты/ЭНСО/Планы/План - 2 - 27.01.2026 - Обновление статусов из YouTrack.md
   
   📈 Прогресс проекта:
   - Было: 10/28 выполнено (36%)
   - Стало: 12/28 выполнено (43%)
   
   📅 Следующий pull сравнит с планом версии 2
   ```

## Формат секции изменений в новом плане

```markdown
## Изменения с версии {N}

| Задача | Issue | Изменение | Кто изменил |
|--------|-------|-----------|-------------|
| Ревью опросника для клиента | [ENSO-11](URL) | Open → Verified (⬜ → ✅) | @manager |
| Отправка опросника Ксении | [ENSO-12](URL) | Open → In Progress (⬜ → 🔄) | @artem |

### Новые комментарии

**ENSO-13 — Формирование КП**
> 👤 @manager (27.01.2026 14:30):
> Добавить в КП блок про анализ звонков

**Вывод:** Ревью опросника завершено. Артём начал отправку опросника клиенту.
```

## Формат истории версий

```markdown
## История версий плана

| Версия | Дата | Изменения |
|--------|------|-----------|
| 1 | 27.01.2026 | Первоначальный план по итогам встречи 2 |
| 2 | 27.01.2026 | Pull из YouTrack: ENSO-11 ⬜→✅, ENSO-12 ⬜→🔄 |
| 3 | 28.01.2026 | Pull из YouTrack: ENSO-12 🔄→✅, ENSO-13 ⬜→🔄 |
```

## Маппинг статусов

| YouTrack | Emoji | Текст |
|----------|-------|-------|
| Open | ⬜ | Не начата |
| In Progress | 🔄 | В процессе |
| Fixed | ✅ | Выполнена |
| Verified | ✅ | Выполнена (проверена) |
| Won't Fix | ⏸️ | На холде |
| Duplicate | ⏸️ | На холде |

**Запрос**: `/yt-project-tasks-pull ЭНСО`

```
1. 🔍 Найден план: План - 1 - 27.01.2026 - MVP готов.md

2. 🔄 Сравнение с YouTrack...

3. 📬 Найдено:
   - 1 изменение статуса: ENSO-11 Open → Verified
   - 0 новых комментариев
   
   → Пользователь: "Да, создать План - 2"

4. ✅ Создан: План - 2 - 27.01.2026 - Обновление статусов из YouTrack.md
   - ENSO-11 теперь ✅
   - Прогресс: 10/28 → 11/28 (39%)
```

### Пример 2: Нет изменений
**Запрос**: `/yt-project-tasks-pull`

```

2. 🔄 Сравнение с YouTrack...

3. ✅ Нет изменений

   Все статусы в YouTrack соответствуют плану версии 2.

```

```

   → Пользователь: "Да, создать план"

   Изменения:
   - ENSO-12 ⬜ → ✅ (Отправка опросника)
   - ENSO-13 ⬜ → 🔄 (Формирование КП)

   Прогресс: 11/28 → 12/28 (43%)
```

## Обработка ошибок

### Нет планов в проекте
```
❌ Планы не найдены

В папке Проекты/ЭНСО/Планы/ нет файлов планов.

Рекомендация:
1. Создайте первый snapshot-план через `/plan-update {PROJECT}`
2. Если нужна публикация в YouTrack — выполните `/yt-project-tasks-push {PROJECT}`
```

### Проект не связан
```
❌ Проект не связан с YouTrack

Проект "ЭНСО" не имеет связи с YouTrack.

Действие: /yt-project-link ЭНСО
```

### Ошибка API
```
❌ Ошибка получения данных из YouTrack

Статус: 401 Unauthorized
Причина: Токен недействителен или истёк

Возможные действия:
1. Проверить YOUTRACK_TOKEN в .env
2. Получить новый токен в YouTrack
```

## API запросы

### Получение всех issues проекта
```
GET /api/issues?query=project:{PROJECT_ID}&fields=id,idReadable,summary,customFields(name,value(name))
Authorization: Bearer {YOUTRACK_TOKEN}
```

### Получение комментариев к issue
```
GET /api/issues/{issueId}/comments?fields=id,text,created,author(login,fullName)
Authorization: Bearer {YOUTRACK_TOKEN}
```

## Алгоритм создания нового плана

1. **Копировать структуру предыдущего плана**
2. **Обновить метаданные**:
   - Версия плана: N → N+1
   - Дата формирования: текущая дата
   - Last Sync: текущий timestamp
   - Sync Direction: pull
3. **Добавить секцию "Изменения с версии N"** после метаданных
4. **Обновить статусы задач** в таблицах
5. **Переместить выполненные задачи** в раздел "Выполненные"
6. **Обновить метрики** (пересчитать проценты)
7. **Добавить запись в "Историю версий"**
8. **Обновить "Резюме текущего статуса"**

---

## Связанные команды
- `/yt-project-link` — связать проект перед pull
- `/yt-project-tasks-push` — исходная публикация задач и обновление Issue ID в плане
- `/plan-update` — актуализировать snapshot-план (если есть новые карточки задач)

---

## Примечания

- Pull не изменяет данные в YouTrack — только читает
- Существующие планы НЕ изменяются — создаётся новая версия
- Каждый plan — snapshot состояния проекта на момент pull
- История планов = история изменений проекта
- Pull можно запускать часто — если нет изменений, новый план не создаётся
- Комментарии из YouTrack добавляются в секцию изменений нового плана
- Если нужно привести карточки `Задачи/*/task.md` в соответствие со статусами YouTrack — предложить сделать это отдельным подтверждаемым шагом после pull
