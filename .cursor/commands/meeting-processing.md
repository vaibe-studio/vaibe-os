# /meeting-processing

> Cursor command wrapper. Canonical skill: `.ai/skills/domain/meeting-processing.md`

## Quick Reference

1. Identify input file (from argument or `Инбокс/`) → **AskQuestion** if multiple → **STOP**
2. Transcribe if audio/video: `python -m tools.meeting_transcriber <file> --lang ru`
3. Analyze transcript → form hypotheses (project, date, topic)
4. **AskQuestion** for project, date, topic, meeting number → **STOP**
5. Generate `summary.md` draft → show → **STOP** for approval
6. Generate `tasks.md` draft → show → **STOP** for approval
7. Show import plan (target folder, files) → **AskQuestion** confirm → **STOP**
8. Execute: create meeting folder, write files

## Execution

Read and follow `.ai/skills/domain/meeting-processing.md` for the full procedure.

## Cursor-specific enhancements

- Use **AskQuestion** for: file selection, project/date/topic, import confirmation
- For transcription: `python -m tools.meeting_transcriber` (see `tools/meeting_transcriber/README.md`)
- Follow `AGENTS.md` → Judgment Boundaries

## Алгоритм

### 1. Определение входного файла
- Если передан путь — использовать его (проверить существование и формат).
- Если путь не передан — искать в `Инбокс/` файлы с расширениями: `.mp4`, `.webm`, `.mp3`, `.wav`, `.md`.
  - Медиа могут быть в `.gitignore` — использовать `ls`/список файлов по факту.
  - Если найдено несколько — показать список и спросить, какой обрабатывать.

**→ СТОП.** Дождаться выбора/подтверждения файла.

### 2. Транскрибация (если аудио/видео)
- Вызвать инструмент транскрибации:
  ```bash
  python -m tools.meeting_transcriber <файл> [--lang ru] [--backend local|assemblyai]
  ```
  По умолчанию использовать `--backend local` (Whisper + pyannote; нужны HUGGINGFACE_TOKEN и GPU). При необходимости можно указать `--backend assemblyai` (облако, ASSEMBLYAI_API_KEY).
- Дождаться завершения. Результат — транскрипт с разметкой спикеров (рядом с файлом или в указанный вывод). Для следующего шага считать, что транскрипт доступен (файл `.md` или содержимое в контексте).
- Если передан уже готовый `.md` транскрипт — шаг транскрибации пропустить.

### 3. Анализ транскрипта
- Прочитать транскрипт и по содержимому сформулировать гипотезы:
  - **Проект**: к какому проекту относится встреча (по названиям, темам, участникам).
  - **Дата**: дата встречи (если упоминается или из имени файла).
  - **Тема**: краткое название встречи на русском.

### 4. Запрос контекста у пользователя (обязательно)
Показать гипотезы и задать вопросы:

- К какому **проекту** относится встреча? (проект должен существовать в `Проекты/`; если нет — предложить создать или выбрать существующий.)
- **Дата** встречи в формате ДД.ММ.ГГГГ?
- **Тема** встречи (краткое название на русском)?
- **Номер встречи** N для папки «Встреча N. ДД.ММ.ГГГГ - Тема» (если в проекте уже есть встречи — подсказать следующий номер; для вставки между встречами без переименования допустим подномер, например 8.1).

**→ СТОП.** Дождаться ответа пользователя.

### 5. Валидация
- Проверить, что папка проекта существует: `Проекты/{PROJECT}/`.
- Дата в формате ДД.ММ.ГГГГ.
- Номер встречи — число или подномер (например 8.1); уникальность в рамках проекта не обязательна, но предложить следующий по счётнику.

### 6. Формирование summary.md
Создать черновик `summary.md` по шаблону (согласованному с `/summarize-meeting`):

```markdown
# Саммари встречи: {ТЕМА}

## Контекст
- **Проект**: {PROJECT}
- **Дата**: {DD.MM.YYYY}
- **Источник**: transcript.md
- **Участники**: по транскрипту (Спикер 1, 2, … или «не указано»)

## Итоги (кратко)
- ...

## Решения и договорённости
- ...

## Риски / блокеры
- ...

## Следующие шаги
- ...

## Открытые вопросы
- ...
```

Содержимое заполнить по анализу транскрипта. Показать черновик пользователю.

**→ СТОП.** Подтверждение или правки.

### 6.1. Формирование tasks.md
Создать черновик `tasks.md` по шаблону (как в `/summarize-meeting`):

- По анализу транскрипта выделить action items, решения с ответственными, следующие шаги.
- Заполнить таблицу: № | Задача | Владелец | Срок | Приоритет | Статус | Примечание. Владельцев и сроки не додумывать — при неясности помечать как гипотезу и спрашивать пользователя.
- Внизу добавить блок «Привязка к vAIbe-os» с подсказкой про `/task-create`.

Шаблон `tasks.md`:

```markdown
# Задачи по итогам встречи: {ТЕМА}

## Список задач

| № | Задача | Владелец | Срок | Приоритет | Статус | Примечание |
|---|--------|----------|------|-----------|--------|------------|
| 1 | ... | ... | ... | 🔴/🟠/🟡/🟢 | ⬜ | ... |

## Привязка к vAIbe-os (опционально)
- Если нужно превратить задачу в полноценную карточку: используйте `/task-create` и вставьте ссылку на `Проекты/{PROJECT}/Задачи/{NUM}-{TITLE}/task.md`.
```

Показать черновик `tasks.md` пользователю. Уточнить: подтверждаем список задач? что исправить (владельцы, сроки, формулировки, приоритет)?

**→ СТОП.** Подтверждение или правки.

### 7. План импорта и подтверждение
Показать план:

- **Целевая папка**: `Проекты/{PROJECT}/Встречи/Встреча {N}. {DD.MM.YYYY} - {TITLE}/`
- **Файлы**:
  - `transcript.md` — транскрипт (перенести/сохранить из шага 2)
  - `summary.md` — саммари (из шага 6)
  - `tasks.md` — задачи по итогам встречи (из шага 6.1)
  - Исходный медиа-файл: оставить в Инбоксе / переместить в папку встречи как `source.{ext}` / переместить в `Проекты/{PROJECT}/Исходные материалы/` / удалить — уточнить у пользователя.
  - Файл транскрипта, сгенерированный инструментом (рядом с исходным или в Инбоксе, например `2026-02-17 14-02-30.md`): после записи в папку встречи удалить / оставить — уточнить у пользователя.
  - **Удалить из Инбокса всё**: по желанию пользователя — удалить после успешного импорта все использованные файлы (исходное медиа: .mp4, .mp3, .wav и т.д., а также сгенерированный .md). Уточнить при плане импорта.

**→ СТОП.** Выполнить импорт только после явного подтверждения.

### 8. Выполнение импорта
- Создать папку встречи, если её нет.
- **Переместить** (через `mv`) сгенерированный `.md`-транскрипт из Инбокса в папку встречи как `transcript.md` — не пересоздавать через Write-инструмент.
- Записать `summary.md` и `tasks.md`.
- Переместить/скопировать/удалить исходный медиа-файл по решению пользователя.
- Если пользователь выбрал «удалить из Инбокса всё» — после записи в папку встречи удалить из Инбокса исходное медиа и сгенерированный .md (если он ещё там; после переноса транскрипта в Инбоксе обычно остаётся только исходное медиа).
- (Опционально) обновить `Проекты/{PROJECT}/README.md` ссылкой на встречу.

### 9. Завершение
- Сообщить пользователю, где сохранены файлы (в т.ч. `tasks.md`).
- (Опционально) предложить создать карточки в `Проекты/{PROJECT}/Задачи/` через `/task-create` для 1–3 ключевых пунктов из `tasks.md`.

## Вызов инструмента транскрибации

Рекомендуемый вызов (из корня репозитория):

```bash
# По умолчанию — local (Whisper + pyannote; нужен HUGGINGFACE_TOKEN и GPU)
python -m tools.meeting_transcriber путь/к/файлу.mp4 --lang ru

# Альтернатива: облако AssemblyAI (нужен ASSEMBLYAI_API_KEY в .env)
python -m tools.meeting_transcriber путь/к/файлу.mp4 --lang ru --backend assemblyai
```

Инструмент описан в `tools/meeting_transcriber/README.md`.

## Обработка ошибок
- **Файл не найден или формат не поддерживается**: сообщить и предложить указать другой путь.
- **Проект не существует**: предложить создать папку проекта или выбрать существующий.
- **Ошибка транскрибации**: вывести сообщение об ошибке; предложить сменить бэкенд (local ↔ assemblyai) или проверить ключи/зависимости (HUGGINGFACE_TOKEN, GPU — для local; ASSEMBLYAI_API_KEY — для assemblyai).

## Связанные команды
- `/summarize-meeting` — саммаризация и задачи по уже сохранённым транскриптам
- `/task-create` — создание карточки задачи из итогов встречи
- `/inbox-check` — разбор файлов из Инбокса (в т.ч. перед обработкой встреч)
